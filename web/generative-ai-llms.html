What is a language model

Autocomplete on steroids

  • A language model assigns probabilities to sequences of words.
  • Its one job: predict the next token, given everything so far.
  • Generate text by doing that repeatedly, feeding each pick back in.
  • Everything else is this, done very well, at scale.

Tokens and embeddings

Words become numbers the model can do math on

  • Text is split into tokens — word-pieces, not whole words.
  • Each token maps to a vector — an embedding.
  • Similar meanings sit nearby in that space.
  • Geometry captures meaning: debit~credit, king~queen.
Embedding space — meaning becomes distance

The 2017 breakthrough

Attention Is All You Need

NeurIPS 2017 · Vaswani, Shazeer, Parmar et al. · Google

Attention Is All You Need

Drop recurrence. Use attention. Process the whole sequence in parallel.
  • No more reading word-by-word — everything at once
  • Scales beautifully on modern GPUs
  • The architecture behind GPT, BERT, Claude, Gemini, Llama

Multi-head attention

Many readers, each tracking one thing

H1

Grammar

Links subjects to verbs, plurals to articles — sentence structure.

H2

Meaning

Connects words by topic — “charge” near “dispute”, “refund”.

H3

Reference

Resolves “her”, “it”, “the account” back to who or what.

Run many heads in parallel, then combine — a richer read than any single one.

3 · Chain-of-thought

Ask it to think step by step

  • For multi-step tasks, request reasoning before the answer.
  • Accuracy improves on anything that needs working-out.
  • Let it say “I don't know” — fewer confident guesses.
  • Trade-off: slower, longer. Use it when it earns its keep.
Think step by step:
1. Fees this month: $12 + $30
2. Last month: $25
3. Change: +$17, a 68% rise
→ Flag: fees up 68%

Pitfalls and safe use

Use it responsibly in a bank

Hallucination

Confidently wrong. Verify facts and figures before you act on them.

Prompt injection

Untrusted text can hide instructions. Don't blindly trust pasted content.

Data privacy

No sensitive customer data into unapproved tools.

Human in the loop

People make the final call on any real decision.

On-premise GenAI

OCBC runs its own open model, in-house

Open-weight · self-hosted
Qwen3.5-35B-A3B
on NVIDIA H200 GPUs · OCBC data centre
  • Open-weights: the parameters are public — we run them ourselves, no external API
  • Mixture-of-experts (A3B): only ~3B of 35B params active per token — strong, yet cheap to run
  • H200: 141 GB of GPU (graphics processing unit) memory, built to serve large models fast

Why on-premise?

  • Data never leaves the bank — privacy, residency, regulation.
  • Full control — fine-tune and guardrail on our own terms.
  • Predictable cost at scale — no per-token API bill.
  • No dependency on an outside provider.
Still a decoder transformer doing next-token prediction — just open-weights, and ours.

Recap

Three things to remember

  • What an LLM is: next-token prediction, trained in 3 stages.
  • The transformer: attention, in parallel, stacked deep.
  • Prompting ladder: clear → examples → CoT → structure → role → chain.

  • It's prediction, not magic.
  • Steer with prompts before fine-tuning.
  • Verify and use responsibly.
