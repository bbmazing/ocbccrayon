The core formula

VRAM for weights = params × bytes

VRAMweights≈parameters×bytes / parameter
FP32
4 bytes
≈ 4N GB
FP16 / BF16
2 bytes
≈ 2N GB
INT8
1 byte
≈ N GB
INT4
~0.5 byte
≈ 0.5N GB
For N billion parameters: ~2N GB in FP16, ~N GB in INT8, ~0.5N GB in INT4 — just the weights.

Don't forget the overhead

Weights aren't everything: add headroom

+

KV cache

the model's memory of the chat so far — grows with context length × batch × layers (KV = key-value)

+

Activations

scratch memory used during each forward pass

+

Runtime

the serving engine itself — about 1–2 GB

Rule of thumb: budget ~1.2–1.5× the weight size for inference — more for long context or large batch. Quantize to fit.

Wrap & Q&A

The whole stack, sized and placed

Open-source & self-hosted — so audio and data stay inside the bank.

What we covered

  • The stack: STT · LLM · embeddings · guardrails · TTS
  • Sizing: ~2N GB FP16 · ~N GB INT8 · ~0.5N GB INT4
  • Overhead: budget ~1.2–1.5× weights for inference
  • GPU: check VRAM first, match the card to the job

The takeaway

  • Quantize and it fits on one 24 GB GPU
  • 70B FP16 needs multiple 80 GB GPUs
  • All open-source & self-hosted — data stays in the bank
