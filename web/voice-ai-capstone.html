Worked example · one model, step by step

Size an 8B LLM in three steps

  • 1 · Weights = params × bytes → 8B × 2 = ~16 GB (FP16).
  • 2 · Spare room ×1.3 for KV cache + activations + runtime → ~21 GB.
  • 3 · Quantize to fit: INT8 → ~11 GB · INT4 → ~6 GB.
  • Same model, ~6–21 GB based on precision.

+ spare room (×1.3)~21 GB
INT8, all-in~11 GB
INT4, all-in~6 GB
The plan for any model: weights → spare room → quantize. That is the whole size skill.

Don't forget the extra memory

Weights aren't all: add spare room

+

KV cache

the model's memory of the chat so far — like a personal notepad of notes per caller; grows with chat length × callers × layers (KV = key–value)

+

Activations

scratch memory used during each pass

+

Runtime

the serving engine itself — about 1–2 GB

Rough guide: plan ~1.2–1.5× the weight size for run time — more for long chats or many callers. Quantize to fit.

KV cache, in plain words

KV cache = a notepad per caller

  • While talking with a caller, the model keeps that caller's notes on a personal notepad — that notepad is the KV cache (key–value cache).
  • Every extra caller at the same time needs their own notepad — so more callers at once need more memory.
  • Weights load once and are shared; the notepads add up per caller.
1 caller
weights once + 1 notepad
3 callers
weights once + 3 notepads
8 callers
weights once + 8 notepads
One shared model, one notepad per live caller — that is why the number of callers at once, not just model size, drives the memory bill.

Worked example · the big one

Why a 70B needs more than one GPU

  • 1 · Weights → 70B × 2 = ~140 GB (FP16) — already past any one card.
  • 2 · Spare room ×1.3 → ~180 GB for KV cache + runtime.
  • 3 · Quantize: INT8 → ~70 GB · INT4 → ~35–40 GB.
  • Even INT4 fills most of one 80 GB card — little room for callers.
FP16 weights (70B × 2)~140 GB
+ spare room (×1.3)~180 GB
INT8, weights~70 GB
INT4, weights~35–40 GB
Split it: FP16 70B spans two or more 80 GB cards (tensor parallel = sharing one model over many GPUs) — that is why 70B is the premium, multi–GPU path.
