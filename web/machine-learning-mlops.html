Classification, intuitively

Classification draws a boundary

  • The model learns a boundary that separates the classes.
  • Anchor: fraud vs legitimate transactions — also alert/clear, approve/decline.
  • New transaction lands on one side → predicted class.
  • Common models: logistic regression, decision trees, random forests, gradient boosting.

How to choose

Which family? Two questions decide

1. Do you have labels?  2. Are you predicting a value, predicting a class, or finding groups?

1

Fraud & AML

Labeled past outcomes, predict a class → supervised classification.

2

Credit / loan approval

Predict a number (risk), then decide → regression + classification.

3

Customer segmentation

No labels, find groups → unsupervised clustering.

4

Product recommendation

Segments + supervised propensity → a blend of both.

Accuracy is a lie

A useless model can score 98%

0

Predict “never fraud” and you are 98% accurate — and catch zero fraud.

Fraud is rare (about 2% of transactions), so always-guess-legit looks great and is worthless. Accuracy hides failure on the cases that matter.

This is why we need precision, recall and the curves that follow.

Train / test split

Never grade a model on data it trained on

  • Hold out a test set the model never sees during training.
  • Grade on the held-out set — that estimates real-world performance.
  • Memorising the training data is not learning. We want generalisation.
  • Typical split: 80% train / 20% test (or k-fold cross-validation).

Precision and recall

Precision vs recall — two questions

  • Precision: of the alerts we raised, how many were real fraud?
  • Recall: of all the real fraud, how much did we catch?
  • High precision = few false alarms. High recall = few misses.
  • You rarely max both at once — next slide.

The trade-off

Move the threshold, trade precision for recall

Alert threshold: 0.50
Drag the cutoff. Raise it → fewer alerts, precision up, recall down. Lower it → catch more, more false alarms.

ROC curve and AUC

ROC curve and AUC

  • Plot true-positive rate against false-positive rate across every threshold.
  • AUC (area under the curve) = ranking quality. 0.5 = coin flip, 1.0 = perfect.
  • Lets you compare models independent of one threshold.
  • On rare events (fraud), prefer the precision-recall curve.
ROC (receiver operating characteristic) — bows to the top-left
OCBC CRAYON · Machine Learning & MLOps
23
Regression metrics

Metrics for predicting a number

MAE

Mean Absolute Error

Average size of the miss, in the same units (dollars). Easy to explain.

RMSE

Root Mean Sq. Error

Like MAE but punishes big misses harder — good when large errors are costly.

R²

R-squared

Share of variance explained. 1.0 = perfect, 0 = no better than the mean.

Use these to evaluate the credit-limit / probability-of-default model behind loan approval.

Overfitting

Overfitting and the bias-variance trade-off

  • Underfit (high bias): too simple, misses the pattern.
  • Overfit (high variance): aces training, flunks reality — it memorised noise.
  • The sweet spot sits in the middle.
  • Check it robustly with cross-validation.
Error vs model complexity

  • ML also depends on data and a learned model, and both change over time.

Monitoring and drift

Models decay — monitor and retrain

  • Data drift: the inputs change (spending patterns shift).
  • Concept drift: the relationship changes (what fraud looks like evolves).
  • Performance silently decays — monitor metrics, alert, retrain.
  • OCBC: stale fraud/AML models, mis-priced credit, less relevant recommendations.

Recap

Three things to remember

  • Workflow spine: Frame → Data → Train → Evaluate → Deploy → Monitor.
  • Metric ladder: confusion matrix → precision / recall → F1 → ROC-AUC.
  • Supervised vs unsupervised decides everything upstream.

Take these away

  • Pick the right metric — accuracy lies on rare events.
  • Beware overfitting — grade on held-out data.
  • A model is a product — it must be monitored and retrained.
